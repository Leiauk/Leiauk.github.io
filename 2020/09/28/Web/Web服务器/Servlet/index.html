Servelet

全称：Server applet

概念：运行在服务器端的小程序

servelet就是一个接口，定义了Java类被浏览器访问（被Tomcat识别）的规则。

使用：定义一个类，实现servelet接口

为什么需要servelet？

因为之所以是动态资源，不同用户的浏览器访问的不一样，所以酒需要有一个特定的条件来使不同的浏览器/用户得到不同的结果。不同的结果需要有不用的内容，不同的内容在java中就是不同的类，类要能被Tomcat识别就需要遵循一定的条件（在这里就是实现servelet接口），浏览器才能够解析它。

实现了servelet的类如何被映射到虚拟路劲供用户访问呢？

  1. 在web.xml中配置servelet。

Servelet的执行原理
  1. 通过url地址，在web’.xml文件中查找demo，在servelet的url-patten中找到demo，再通过demo的serveletclass中找到资源。
    1. Tomcat将全类名对应的字节码文件加载入内存，Class.forname（）
    2. 通过反射class.newInstance()创建对象。
    3. 调用类的service方法

  • 当服务器接收到客户端浏览器的请求后，会解析请求url的路径，获取访问servlet的资源路径
  • 查找web.xml文件是否有对应的标签体内容
  • 如果有，则再找到对应全类名
  • tomcat会将字节码文件加载进内存，并且创建对象
  • 调用其方法
Servlet的方法与生命周期

  1. Servlet的方法：

    1. 初始化方法：servlet被创建时执行，只会执行一次。

      1. 指定servlet的创建时间

        10默认是-1（第一次访问是被创建），在服务器启动时被创建置值为正值。

      2. servlet在运行时只创建一次，说明servlet是单例的。所以在servlet的过程中，是线程不安全的。

        • 解决方法：尽量不要在servlet中定义成员变量，如果不得已定义了成员变量，不要同时对其进行修改。（并发修改避免）

    2. 提供服务方法：每一次servlet被访问时，都会执行

    3. 销毁方法：servlet被杀死时（服务器正常关闭）时被执行，只会执行一次。

      1. 只用服务器正常关闭时才会执行servlet
      2. destroy方法在servlet被销毁时执行，一般用于销毁资源。

    4. 获取servletconfig的配置对象

  1. 获取servlet的一些信息：版本，作者等

Servlet：
  1. 创建JavaEE项目
  2. 定义一个类，实现servlet接口
  3. 实现接口中的抽象方法
  4. 配置servlet
配置servlet更加简单的方法：
  • 直接在servlet类上添加注解@WebServlet(“/demo1”)
IDEA与Tomcat
  1. IDEA与Tomcat部署的项目单独建立一份配置文件
    • 查看控制台的log：tomcat部署web项目的路径
  2. 工作空间项目 Tomcat部署的Web项目
    • Tomcat真正访问的是 “Tomcat部署的Web项目”；“Tomcat部署的Web项目”对应着“工作空间项目”下的所有Web资源。
    • WEB-INF目录下的资源不能被浏览器直接访问
  3. IDEA中Tomcat中输出日志出现中文乱码问题
    • 解决方法：找到C:\leiauk\Tomcat\apache-tomcat-8.5.57-windows-x64\apache-tomcat-8.5.57\conf\logging.proprties 文件，
    • java.util.logging.ConsoleHandler.encoding = UTF-8修改为java.util.logging.ConsoleHandler.encoding = GBK
Servlet的体系架构

  1. Servlet—接口

  2. GenericServlet –抽象类

    1. 将servlet接口中其他的方法做了默认的空实现，只将servlet方法作为抽象
    2. 需要其他的方法自行添加即可

  3. HttpServlet–抽象类

    1. 产生的原因：因为servlet中的在获取资源/返回资源的时候需要判断Http的请求方式，这会显得十分繁琐。所以在HttpServlet中将此步骤给处理了。

      1. 将其分解为doGet()与doPost();

    2. HttpServlet是对Http协议的一种封装。

ServletContext

  1. 概念：代表整个web应用，可以与程序的容器(服务器)来通信。

  2. 获取：

    1. 通过request对象获取

      request.getServletContext()

    2. 通过HttpServletContext()获取

      this.getServletContext()

  3. 功能：

    1. 获取MIME类型

      1. 概念：在互联网通信过程中定义的一种文件数据类型
      2. 格式：大类型/小类型 text/html image/jpeg
      3. 用于context-type中的类型设置：用于数据的传输
      4. 获取：String getMineType(String name )

    2. 域对象：共享数据

      1. setAttribute(String name;object value)
      2. getAttribute(Sttring name)

    3. 获取文件的真实(服务器)路径

      String getRealPath(String Path)

  2. 定义Servlet

    3. 指定response响应头：Context-disposition;attachment;filename=xxx
