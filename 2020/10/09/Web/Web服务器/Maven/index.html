Maven

  1. 概念：Maven集成打包的软件，IDEA中存在内置的Maven。
  2. 生命周期：

典型的 Maven 构建（build）生命周期是由以下几个阶段的序列组成的：

阶段 处理 描述
验证 validate 验证项目 验证项目是否正确且所有必须信息是可用的
编译 compile 执行编译 源代码编译在此阶段完成
测试 Test 测试 使用适当的单元测试框架（例如JUnit）运行测试。
包装 package 打包 创建JAR/WAR包如在 pom.xml 中定义提及的包
检查 verify 检查 对集成测试的结果进行检查，以保证质量达标
安装 install 安装 安装打包的项目到本地仓库，以供其他项目使用
部署 deploy 部署 拷贝最终的工程包到远程仓库中，以共享给其他开发人员和工程

    1. maven中添加本地的maven目录

    2. 设置所需的配置文件

    3. 添加vm option ：为了第二次创建maven项目的时候不需要重新去网络上download新的资源插件，而是先从本地检索。

    4. 需要了解maven版本与java版本匹配问题

    5. 在idea中不会为我们自动创建maven的资源目录，如果选择idea版本的话，不会进行相关的创建，而如果不选择版本的话，默认的创建的项目中就包含了资源文件resource。而是创建对应的文件夹之后形成将其并列为资源目录，同样测试文件中也是如此的。

    6. 在pom.xml文件中导入jar包的时候需要看本地是否有对应的jar包，可以将常用的jar包放在本地中，这样方便去开发效率，如果没有的话可以百度搜索maven中央仓库，导入相关的依赖（代码形式）当联网的时候就可以下载中央仓库相关的jar包了。

    7. 在编译的过程中，tomcat：run的过程中如果点击了

      就会变成离线模式，在运行的过程中如果是需要下载相关的资源就需要将离线模式关闭，不然就会报错，

      出现错误：C:\leiauk\tools\JDK9\JDK\bin\java.exe -Dmaven.multiModuleProjectDirectory=C:\leiauk\IDEA\Maven_webapp -Dmaven.home=C:\leiauk\IDEA\plugins\maven\lib\maven3 -Dclassworlds.conf=C:\leiauk\IDEA\plugins\maven\lib\maven3\bin\m2.conf -Dmaven.ext.class.path=C:\leiauk\IDEA\plugins\maven\lib\maven-event-listener.jar -javaagent:C:\leiauk\IDEA\lib\idea_rt.jar=57482:C:\leiauk\IDEA\bin -Dfile.encoding=UTF-8 -classpath C:\leiauk\IDEA\plugins\maven\lib\maven3\boot\plexus-classworlds-2.6.0.jar org.codehaus.classworlds.Launcher -Didea.version2019.3.3 -s C:\leiauk\Maven\apache-maven-3.6.3\conf\settings.xml tomcat:run

      找不到网页，是因为IDEA中maven内置的tomcat版本是6的低版本，需要在设置

中将其设置为7的版本。

运行指令tomcat7:run即可运行成功。
