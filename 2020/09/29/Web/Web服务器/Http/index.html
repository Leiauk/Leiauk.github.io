HTTP

Hyper Text Transfer Protocol 超文本传输协议

  • 特点：
    1. 基于TCP/IP协议
    2. 默认端口：80
    3. 基于请求响应模型
      • 响应模型：一次请求一次响应
    4. 无状态的：每次请求都是分开的
  • 历史版本
    • 1.0：在请求资源的时候每次请求资源都会断开与服务器的链接，再次请求就需要重新地建立链接。–消耗资源与时间
    • 1.1：请求得到响应后等待一段时间看是否还要资源的请求，这样就形成了对之前链接的复用。

请求消息

  1. 数据格式

    1. 请求行

      • 请求方式 请求url 请求协议/版本

      （GET / login.html HTTP/1.1）

      1. 请求方式
        • 7种请求方式（常见GET POST）
          1. GET：
            1. 请求参数在请求行中，在url后。
            2. 请求的url的长度是有限制的
            3. 不太安全
          2. Post：
            1. 请求参数在请求体中
            2. 请求的url长度是没有限制的
            3. 相对安全

    2. 请求头

      • 请求名称：请求头值

        1. 常见的请求头

          1. User-Agent：浏览器告诉服务器所使用的浏览器版本信息–可以在服务器端获取该头的信息，解决浏览器的兼容问题

          2. Referer：http://localhost/login.html：告诉服务器当前请求从哪里来

            1. 作用：

              1. 防盗链

              2. 统计工作

  1. 请求空行

    • 空行：用于分割请求头与请求体

  2. 请求体（为Post方式时才会有请求体：为Post的参数）

    • 封装POST请求消息的请求参数
Request

  1. request与response对象的原理

    1. request和respondse对象是由服务器创建的
    2. request对象是来获取请求消息的，response对象是来设置响应消息

  2. request对象得继承体系结果

    1. ServletRequest–接口

      ​ |继承

    2. HttpServletRequest –接口

      ​ |实现

    3. org.apache.catalina.connector.RequestFacade –类（由Tomcat创建）

  1. request功能：

    1. 获取请求的数据

      1. 获取请求行数据

        GET /day14/demo?name=huangjia HTTP/1.1

        1. 获取请求方式:GET

          String getMethod()

        2. 获取虚拟目录：/day14

          String getContextPath()

        3. 获取Servlet的路径：/demo

          String geServletPath()

        4. 获取get方式的请求参数:name=huangjia

          String QueryString()

        5. 获取请求URI:统一资源标识符

          String getRequestURI()::/day14/demo

        6. 获取请求URL：统一资源定位符

          String getRequestURL():Http://localhost/day14/demo

        7. 获取协议版本:HTTP/1.1

          String getProtocol()

        8. 获取客户机的IP地址

          String getRemoteAddr()

      2. 获取请求头数据

        1. 方法
          • String getHeader(String name):通过请求头的名称获取请求头的值
          • Enumeration getHeaderName()获取所以的请求头名称

      3. 获取请求体数据

        1. 请求体：只用POST请求方式才有请求体，在请求体种封装了POST请求的请求参数
        2. 获取请求体的步骤
          1. 获取流对象
            • bufferedReader getReader():字符输入流
            • ServletInputStream get InputStream()：字节输入流
          2. 从流对象种获取数据

    2. 其他功能

      1. 获取请求参数通用方式:
        1. String getParameter(String name):根据参数的名称获取参数值
        2. String[] getParameterValues(String name):根据参数名称获取参数值的数组
        3. EnumerationgetParameterNames();获取所以请求的参数名称
        4. Map<String,String[]>getParameterMap():获取所以参数 的map集合
          • 因为获取方式是通用的，所以将get与post想乎调用，get中调用post/post中调用get。
      2. 请求转发：一种在服务器内部的资源跳转方式
        1. 步骤：
          1. 通过request对象获取请求转发器对象：requestDispatcher getRequestDispatcher(String path)
          2. 使用RequestDispatcher对象进行转发：forward(ServletRequest request,ServletResponse response)
        2. 特点：
          1. 浏览器地址栏的路径不发生变化
          2. 只能转发当前服务器中的内部资源
          3. 转发是一次请求
      3. 共享数据
        1. 域对象：一个有作业范围的对象，可以在范围内共享对象。
        2. request范围：代表了一次请求的 范围，一般用于请求转发的多个资源中共享数据
        3. 方法：
          1. void setAttribute(String name,Object obj);存储数据
          2. Object getAttribute(String name);通过键获取值
          3. void removeAttribute(String name)通过键值移除对象
      4. 获取ServletContext
        • servletContext getServletContext()

    响应消息

    1. 响应过程

      1. Tomcat会根据url中的资源路径，创建对应的Servlet对象
      2. Tomcat服务器随后创建request与response对象。request对象中封装请求消息的数据
      3. Tomcat将request和response对象传递给server方法，并调取server方法
      4. 可以通过request对象获取请求消息的数据，response对象设置对应的响应数据
      5. 服务器在给浏览器做出响应之前，从response对象中传入开发人员设置的响应消息数据

    2. 数据格式

      1. 响应行
        1. 组成：协议/版本 响应状态码 状态码描述
        2. 响应状态码
          • 分类
            1. 1xx：服务器接收客户端消息但没有接收完成，等待一段时间之后，发送1xx状态码咨询服务器是否还有消息发送
            2. 2xx：成功
            3. 3xx:重定向。代表：304（访问缓存）；302（重定向）
            4. 4xx：客户端错误。代表：404（请求路径没有对应的资源）405（请求方式没有对应的doXxx方法）
            5. 5xx：服务器端错误。代表：500（服务器内部出现异常）
      2. 响应头
        1. 格式：头名称：值
        2. 常见的响应头
          1. Context-Type：服务器告诉客户端本次响应体数据格式以及编码格式
          2. Context-disposition：服务器告诉浏览器以什么格式打开响应体数据
            • in-line：默认值，在当前页面打开
            • attachment ；filename=xxx：以附件形式打开响应体。文件下载
      3. 响应空行
      4. 响应体：传输的数据

    3. response对象

      1. 功能：设置响应消息

        1. 设置响应头

          格式：HTTP/1.1 200 OK

          • 设置状态码：setStatus(int sc)

        2. 设置响应行setHeader(String name,String value)

        3. 设置响应体

          • 使用步骤
          1. 获取输出流
            1. 字符输出流PrintWriter getWriter()
            2. 字节输出流ServletOutputStream getOutputStream()
          2. 使用输出流，将数据输出到客户端浏览器

      2. 重定向：资源跳转的方式

        1. 实现方法

          1. 方法一
            1. 设置状态码setStatus(302)
            2. 设置响应头location:response.setHeader(“location”,”url路径”)
          2. 方法二
          • 直接设置response.sendRedirect(“url路径”)

        2. 重定向特点redirect

          • 地址发生变化
          • 重定向可以访问其他站点(服务器)的资源
          • 重定向的两次请求，不能使用request对象来共享数据

        3. 转发的特点 forward

          • 转发地址栏不变
          • 转发只能访问服务器下的资源
          • 转发是一次请求，可以使用request对象来共享数据

          （forward与redirect的区别）

相对路径与绝对路径

当是给客户端浏览器使用时，需要添加虚拟目录(绝对路径)，当是给服务器使用时，不需要添加虚拟目录(相对路径)。其根本的依据是判断对应的请求是谁需要的。其中，如果是需要添加虚拟目录，一般将路径获取为变量，通过变量来动态的赋值。
