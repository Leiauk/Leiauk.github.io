redis

  1. 概念：redis是一款高性能的NOSQL系列的非关系型数据库

    非关系数据库：存储为Key-Value，数据之间没有关联关系，数据存储在内存中（redis）。因为存储在内存中，所以查询速度上更加的快与便捷。非关系数据库的不同系统（redis||hbase）的差别是十分大的。一般会将数据存储在关系型数据库中，在nosql数据库中去备份存储关系型数据库的数据。

    主流的NOSQL产品：

    1. 键值存储数据库：Tokoy,Cabinet/Tyrant，Redis，Berkeley，DB

      应用：内存缓存，主要用于处理大量数据的高访问负载

      缺点：存储的数据缺少结构化

      优势：快速查询

    2. 列存储数据库

      Cassandra,,HBase，Riak

      优点：进行分布式存储

      劣势：功能相互局限

    3. 文档型数据库

      CouchDB，MongoDB

      缺点：查询性能不高，而且缺乏统一的查询语句

      优势：数据结果要求不严格

    4. 图形数据库

      Neo4J,InfoGrid,Infinite,Graph

      缺点：需要对整个图计算才能得出结果，不容易做分布式的集群方案

      优点：利用图结构相关算法

      Redis是一个C语言开发的键值对数据库，Redis用不同的键值对象来存储不同的存储需求，目前支持的类型如下：

      字符串String，哈希类型hash，列表类型list，集合类型set，

  2. 下载安装

    1. 下载完直接解压就可以使用

      • redis.window.conf配置文件

      • redis-cli.exe:redis客户端

      • redis-server.exe：redis服务器

        文件下载过程注意看文件是否是正确的，如果出现运行的redis-service 不是可执行文件，可能是下载的文件错误了。

  3. 命令操作

    1. 数据结构
      1. redis存储的是键值对
      2. key都是字符串，value值可以有五种不同的结构
        • String
        • hash，map格式
        • list：linkList格式
        • set:hashSet格式
        • sortedset：有序集合，自然排序且不允许重复。

  4. 持久化操作

    1. 字符串类型

      1. 存储：Set key value
      2. 获取：get key
      3. 删除：delete

    2. 哈希类型

      1. 存储:hset key field value

      2. 获取：hget key field

        hgetall key :获取所以的键值对

      3. 删除：hdel key field

    3. 列表类型

      1. 添加
        1. lpush key value：将元素加入列表左表
        2. rpush key value：将元素加入列表右表
      2. 获取
        • lrange key start end：通过范围获取 例如：lrange mylist 0 -1 ：获取mylist中的所以元素
      3. 删除
        1. lpop key：删除列表最左边的元素，并将元素返回
        2. rpop key：删除列表最右边的元素，并将元素返回

    4. 集合类型

      1. 存储：sadd key value
      2. 获取：smembers key：获取set集合中所以元素
      3. 删除：srem key value 删除set中的元素

    5. 有序集合类型

      1. 存储 ：zadd key score value
      2. 获取：zrange key start end
      3. 删除：zren key value

    6. 通用命令

      1. keys*：查询所以键
      2. type key：获取对应键的类型
      3. del key：删除对应键值对

    7. redis是一个内存数据库，服务器重启会自动丢失数据，我们可以将数据存储到硬盘中。

      • RDB（文件后缀.rdb）：默认方式，不需要进行配置，默认使用这种机制。在一定的间隔时间中，检测key的变化情况，然后持久化存储数据。

        • 步骤：

          1. 编辑redis.windows.conf文件

            save 900 1 在15分钟内有key值的改变就会持久化存储一次

            save 300 10 在5分钟后有10个key值改变就持久化存储一次

            save 60 10000 在1分钟内有10000个key值概念就持久化存储一次

          2. 重新启动redis服务器，并指定配置文件

      • AOF：日志记录的方式，可以记录每一条命令的操作，每一次命令操作后来持久化数据。

        1. 编辑配置文件

          appendonly yes ：默认为NO关闭，修改为开启YES

          appendonly awayls：每一次操作都进行持久化

          appendonly everysec：每一秒进行一次持久化操作

          appendonly no：不进行持久化操作

  5. 使用java客户端操作redis

    1. Jedis：一款java操作redis数据库得工具，JDBC工具。

    2. 用于缓存操作，实现访问速度的提升。
