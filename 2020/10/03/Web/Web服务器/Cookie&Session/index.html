会话技术

  1. 相关概念

    • 会话：一次会话中包含多次请求与响应
    • 一次会话：浏览器第一次给服务器资源发视请求，会话建立，直到一方断开为止。

  2. 功能：在一次会话的范围内的多次请求间，共享数据

  3. 方式：

    1. Cookie:客户端会话技术

      • 概念：将数据保存到客户端的技术

      • 快速入门

        1. 创建cookie对象，指定数据

          new cookie(String name,String value)

        2. 发送cookie对象

          response.addCookie(Cookie cookie)

        3. 获取cookie，拿到数据

          Cookie request.getCookies()

        4. 相关的细节

          1. 一次可不可以发送多个cookie？

            可以创建多个cookie对象，使用response调用多次addCookie方法发送cookie即可

          2. cookie在浏览器中保存多长时间？

            默认情况下，当前浏览器关闭后，Cookie数据被销毁，如果需要持久化的存储，需要通过setMaxAge(int seconds)。【默认为负数，当为0时，删除cookie信息】

          3. cookie能不能存中文？

          4. cookie获取范围多大？

  1. Session：服务器端会话技术

    1. 概念：服务端会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器的对象中。HttpSession

    2. 快速入门：

      1. 获取HttpSession对象

        HttpSession request.getSession();

      2. 使用HttpSession对象

        object getAttribute(String name)//获取数据

        void setAttribute(String name,Object value)//存储数据

        void removeAttribute(String name)

      3. 原理:

        Session是依赖于Cookie的

      4. 细节：

        1. 当客户端关闭后，服务器不关闭，两次Session 是否为同一个？

          默认情况下，不是。如果需要让其访问相同，可以借助Cookie来存放Session。

    2. 客户端不关闭，服务器关闭，两次获取的Session是否为同一个？

        不一样，ID是不相同的， 但是要确保Session数据不丢失

        Session的钝化：

        ​    在服务器正常关闭的时候，将Session对象序列化到硬盘上

        Session的活化：

        ​    在服务器启动后，将Session文件转化为内存的session对象即可

    3. Session的失效时间？

        1. 服务器关闭
        2. 调用关闭方法:session对象调用invalidate()
        3. session的默认失效时间：30分钟

5. session的特点：

    1. session用于存储一次会话的多次请求的数据，存在服务器端
    2. session可以存储任意类型，任意大小的数据

6. session与cookie的区别

    1. session存储数据的服务器，cookie在客户端
    2. session没有数据大小限制，cookie有
    3. session数据安全，cookie相对不安全
JSP

  1. 概念：Java Server Pages :Java服务器端页面

  2. 特点：同时支持Html与Java代码；更加方便的将静态页面与Java逻辑代码相互结合。

  3. 原理：JSP本质上是一个Servlet

  4. JSP的脚本：

    1. <% Java代码%>：定义的java代码，在service方法中可以定义什么，该脚本就可以定义什么。
    2. <%! Java代码%>：在JSP转换后的java类的成员位置
    3. <%=Java代码%>：会输出在页面上，输出语句中定义什么，该脚本就可以定义什么。

  5. JSP内置对象

    1. 特点：无需获取与创建，可以直接使用的对象。

    2. 共9个内置对象

      request HttpServletRequest 一次请求访问的多个资源（转发）
      response HttpServletResponse 响应对象
      out Jsp.Write 字符输出流对象，可以将数据输出到页面上
      pageContext PageContext 当前页面的数据
      session HttpSession 一个会话的多个请求
      application ServletContext 当前用户间的共享数据
      page Object 当前页面（servlet）的对象 this
      config ServletConfig servlet的配置对象
      exception Throwable 异常对象 
out与response.getWrite()类似。response.getWrit()与out.Write()的区别：在Tomcat服务器真正给客户端做响应之前，会先找到response缓冲区数据，再找out缓冲区数据。response.getWrite()数据输出永远在out.Write()之前。

1.

  1. 指令：

    1. 作业：配置jsp页面，导入资源文件

    2. 格式

      1. <%@ 指令名称 属性名=属性值…%>

    3. 分类：

      1. page ：配置jsp页面

        contentType：等同于response.setContentType()

        • 设置响应体的mime类型以及字符集
        • 设置当前jsp页面的编码

        import：导包

        errorPage：当前页面发送异常后会自动跳转到指定的错误提示页面

        isErrorPage：Boolean值；标识 当前页面是否为错误提示页面。

        ​ true:是错误提示页面，可以使用内置对象exception

        ​ flase：不能使用exception

      2. include：页面包含的。导入页面的资源文件

        <%@include file=”top.jsp”%>

      3. taglib：导入资源

        <%@taglib prefix=”c” uri=”http://java.sun.com/jsp/jst1/core"%>

        prefix:前缀，自定义的

  2. 注释

    1. html注释：

      <!– – >:只能注释Html代码片段

    2. jsp注释

      <%– –%>

MVC：开发模式

  1. jsp演变历史

    1. 早期只有servlet，只能使用response输出标签数据，非常麻烦
    2. 后来出现了JSP进一步解决的麻烦的问题，但是因为HTML代码与JAVA代码的相互嵌套，使得代码的可读性变得很差，从而影响到软件的维护，难于分工协作。
    3. 到后来，web开发借鉴MVC的开发模式，是的程序更加合理。

  2. MVC（开发模式 不是设计模式）：

    1. Model

      完成具体的业务操作；如：查询数据库，封装对象

    2. View

      展示数据

    3. Controller

      • 获取用户的输入

      • 调用模型

      • 将数据交给视图

  3. EL表达式

    1. 概念：expression language表达式语言

    2. 作用：替换和简化jsp页面中的java代码的编写

    3. 语法：${表达式}

  1. 注意点:

    • jsp默认是支持EL表达式的，如果要忽略EL表达式
      1. 设置jsp指令中的isELIgnored=”true”:忽略当前jsp页面中的EL表达式
      2. $(表达式)：忽略当前这个EL表达式

  2. 使用

    1. 获取值

      1. EL表达式只能从域对象中获取值

        语法：$(域名称.键名)：从指定域中获取指定键的值

        pageScope —>pageContext

        requestScope —>request

        sessionScope —>session

        applicationScope —>application(ServletContext)

      2. 举例：在request域中存取了name=张三

        ​ 获取张三：${requestScope.name}

      3. ${键名}：表示依次从最小的域中查找是否有该键对应的值，直到找到为止。

        ${name};遍历域对象进行查找，查找顺序为从小范围到大范围。

    2. 获取对象，List集合，Map集合

      1. 对象：${域名称.键名.属性名}

        本质上回去调用对象的getter方法

      2. List集合：${域名称.键名[索引]}

      3. Map集合：${域名称.键值}||${域名称[键值]}
