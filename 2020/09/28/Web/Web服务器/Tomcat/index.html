Web相关的概念

  1. 软件架构
    • C/S 客户端-服务器
    • B/S 浏览器-服务器
  2. 资源分类
    1. 静态资源
      1. 所有的用户访问后，得到的结果都是一样的，都称为静态资源，静态资源能够被浏览器直接解析，因为浏览器中内置了静态资源解析器。
      2. Html–CSS–Javascript
    2. 动态资源
      1. 每个用户得到的结果可能不一样，称为动态资源，动态资源被访问后，需要先转化为静态资源，再返回给浏览器。（响应给浏览器）
      2. servlet/jsp，php，asp…
      3. Java动态项目的目录结构
        • 项目的根目录
          • WEB-INF目录
            • web.xml：web项目的核心配置文件
            • classes目录：字节码文件的目录
            • lib目录：依赖jar包
  3. 网络通信的三要素
    1. ip：电子设备（计算机）在网络中的唯一标识
    2. 端口：应用程序在计算机中的唯一标识。范围在0-25536
    3. 传输协议：规定了数据传输的规则
      1. 基础协议
        1. Tcp：安全协议。速度比udp慢
        2. udp：不安全协议。速度快
  4. web服务器
    1. 服务器：安装了服务器软件的计算机
    2. 服务器软件：接收用户的请求，处理需求，做出响应
    3. web服务器软件：接收用户的请求，处理需求，做出响应
    4. 常见的java相关的web服务器软件
      1. webLogic：Oracle公司，大型的JavaEE服务器，支持所有的JavaEE规范
      2. WebSphere，IBM公司，大型的JavaEE服务器，支持所有的JavaEE规范
      3. JBOSS：JBOSS公司的，大型的JavaEE服务器，支持所有的JavaEE规范
      4. Tomcat：Apache基金公司的，中小型的JavaEE服务器，仅仅支持少量的JavaEE规范的servlet/jsp。开源免费。
        • JavaEE：Java语言在企业级的开发中使用的技术规范的总和，共13项大的规范。
      5. 在web服务器软件中，可以部署web项目，让用户通过浏览器来访问这些项目
  5. Tomcat

    1. Tomcat的安装

      • 直接解压
      • 安装目录不要有中文

    2. tomcat的启动

      1. bin目录中的startup.dat（windows）startup.sh(linux)—自带的项目

        • .bat是window的批处理文件

      2. 打开后窗口不可以关闭

      3. 访问服务器===ip地址+tomcat端口号

      4. 可能遇到的问题

        1. 点击startup后窗口一闪而过

          • 原因：没有正确配置JAVA_HOME的环境变量，因为在.dat文件中使用的是JAVA_HOME文件路径。

        2. 启动报错

          1. 原因：Tomcat端口号的被占用了。（默认的端口号是8080）

            • 通过端口开查看进程的命令：netstat -ano

          2. 解决方法

            1. 直接杀死占用端口的进程：在任务管理器通过pid杀死进程，任务管理器-选项-显示列-pid.将占用端口的进程杀死。

            2. 修改端口号：conf/server.xml文件修改connector与redirectprot等的port端口号

              connector的port是默认的端口号。一般会将Tomcat的默认端口号修改为80；80 端口号是Http协议 的默认端口号，在访问的时候就不需要输入端口号。

  1. Tomcat的关闭

    1. 正常关闭
      • bin/shutdown文件
      • cmd窗口中ctrl+c
    2. 点击窗口关闭

  2. Tomcat的配置

    1. 部署项目的方式：

      1. 直接将项目放置在webapps文件夹即可

      2. 简单部署

        1. 当有多个文件的时候，将html文件直接打包为zip文件，然后修改zip 的后缀为war即可完成多文件的部署。（最好是用专门的war文件打包工具）

          • 浏览器的解析器会自动的解压缩，并解析。

        2. 配置conf/server.xml文件

          1. 在标签体中配置

          2. <context doBase="D:hello" path="/hehe"/>
         1. doBase:项目存放的路径
         2. path：虚拟目录
         3. 这种方法是在整个项目的配置文件中修改的，可能会引起不必要的麻烦：
            - 误删了某些文件/其他文件无法进行解析等问题
            - 每次的部署都需要重启服务器
3. 解决2中存在的问题：在conf/catalina/localhost中创建任意名称的XML文件，在文件中编写<context doBase="D:hello" />
      - 虚拟目录：xml文件的名称
     - 这种方式称为热部署的方式，如果不需要改部署文件了，只需要删除该文件/加入文件后缀.bak
        - .bak代表的是临时文件
        - 每次部署不需要重启服务器

  1. Tomcat的目录

    • bin 可执行文件
    • conf 配置文件
  • lib 依赖jar包
    • logs 日志文件
  • temp 临时文件
    • webapps 存放web项目
    • work 存放运行时的数据

  1. Tomcat简单集成到IDEA中

    1. 设置中添加Tomcat

  2. 新建项目/模块选择JavaEnterprise

    1. 选择对应的JavaEE版本与Tomcat版本

  3. 打勾WebApplication

    1. 选择version版本（与JaveEE版本相互匹配）
    2. Create web.xml打勾

  4. 创建成功后的web项目文件

  • src
    • web
      • WEB-INF
        • web.xml（里面包含的是web项目的配置信息）
      • index.jsp (默认页面文件)

  1. 更新资源时就重新部署-IDEA设置

    1. 在RUN/DEbug configuration 中 on uodate action 选址 update resources.
    2. 在RUN/DEbug configuration 中 on frame deactivation 选址 update resources.
